## SUPPORT TIER STRUCTURE

Sarang provides support through a tiered model to manage scale without unsustainable obligations:

### Tier 0 — Self-Service (Always Available)
Resources users can access without contacting Aszurex:
- In-app Help documentation and tooltips
- Knowledge base articles (when published)
- FAQ pages on Aszurex website
- Video tutorials (when published)
- GitHub Issues for bug reports

### Tier 1 — Community / Best-Effort Support
- GitHub Issues and Discussions
- Aszurex contact email
- Response time: Best effort, no SLA or guaranteed timeline
- Scope: Bug reports, feature requests, general usage questions

### Tier 2 — Commercial Services (Separate Engagement Required)
Services beyond free software support, requiring a separate commercial contract:
- Implementation and setup assistance
- Data migration services
- Custom feature development
- Staff training and onboarding workshops
- Dedicated support contracts with SLAs
- Custom integrations with third-party systems

## ESCALATION PROTOCOL FOR USERS

1. **Self-diagnose:** Check in-app Help or FAQ first
2. **Known issues:** Search existing GitHub Issues
3. **Bug report:** Submit via GitHub Issues with: version number, OS version, steps to reproduce, screenshots, expected vs. actual behavior
4. **Urgent data issue:** Email Aszurex with "URGENT" in subject — include: version, OS version, error reference code (from ERROR_CATALOG.md), backup file if safe to share
5. **Feature request:** GitHub Discussions with label "enhancement"

## BUG SEVERITY CLASSIFICATION

| Severity | Definition | Response Priority |
|----------|-----------|------------------|
| Critical | Data loss or corruption risk | Before next release — may require hotfix |
| High | Core workflow completely blocked | Next release |
| Medium | Feature broken but workaround exists | Next planned release |
| Low | Minor visual or usability issue | Batch with regular release |
| Enhancement | New feature request | Evaluated against roadmap |
