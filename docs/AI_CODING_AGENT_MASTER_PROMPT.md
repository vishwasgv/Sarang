## QUALITY GATES CHECKLIST

Before marking any implementation task complete, verify all of the following:

### Security Gate
- [ ] `contextIsolation: true`, `sandbox: true`, `nodeIntegration: false` on ALL BrowserWindows
- [ ] `enableRemoteModule: false`, `webviewTag: false`, `experimentalFeatures: false`
- [ ] Every new IPC handler calls `requirePermission()` before any logic
- [ ] No renderer process code directly imports Prisma
- [ ] No `eval()`, `Function()`, `dangerouslySetInnerHTML`
- [ ] Input validation with Zod on all IPC handler parameters

### Payment Gate (PM005)
- [ ] No code verifies or polls payment status
- [ ] No code claims payment success
- [ ] No integration with payment gateways
- [ ] UPI QR code only generates the image — does NOT confirm receipt

### Data Integrity Gate
- [ ] All financial calculations done in service layer, not in React components
- [ ] Tax amounts stored as precise values, not recalculated at display time
- [ ] All database mutations wrapped in Prisma transactions where multiple records change
- [ ] Inventory movements created for every invoice line item

### Code Quality Gate
- [ ] All new services return `ServiceResult<T>` (not raw throws to caller)
- [ ] All errors use codes from `ERROR_CATALOG.md`
- [ ] No `any` TypeScript type without explicit justification
- [ ] No `console.log` in production paths — use structured logger
- [ ] All public service functions have corresponding unit tests

### Audit Gate
- [ ] Every sensitive operation creates an audit log entry
- [ ] Audit logs include: userId, action, entity, entityId, timestamp, before/after values
- [ ] Audit logs are never deleted — soft system only

### Privacy Gate
- [ ] No network requests to external servers added
- [ ] No new telemetry, analytics, or tracking added
- [ ] `publish: null` in electron-builder config unchanged
- [ ] No credentials, API keys, or secrets in code or config

## IMPLEMENTATION QUICK REFERENCE

### Correct IPC Handler Pattern
```typescript
// ipc/billing-handlers.ts
ipcMain.handle('billing:createInvoice', async (event, data) => {
  requirePermission(event, 'billing.create');           // ALWAYS first
  const validated = createInvoiceSchema.parse(data);    // ALWAYS validate
  return billingService.createInvoice(validated);       // Returns ServiceResult<Invoice>
});
```

### Correct Service Pattern
```typescript
// services/billing-service.ts
export async function createInvoice(data: CreateInvoiceInput): Promise<ServiceResult<Invoice>> {
  try {
    const result = await prisma.$transaction(async (tx) => {
      // ... all mutations inside transaction
    });
    return { success: true, data: result };
  } catch (error) {
    logger.error('billing:createInvoice', error);
    return { success: false, error: { code: 'INV-001', message: 'Failed to create invoice' } };
  }
}
```

### Tax Calculation Pattern (CORRECT)
```typescript
// In service layer only
function calculateLineTax(unitPrice: number, quantity: number, taxRate: number): number {
  return Math.round((unitPrice * quantity * taxRate / 100) * 100) / 100;
}
```
